## 二、核心功能模块

### 2.1 主窗口（Main Window）

类 VSCode 风格的笔记编辑界面，是应用的主体。

- **笔记列表面板**：左侧展示所有笔记，平铺排列（暂不支持文件夹/标签分类）
- **编辑器面板**：右侧为 Markdown 编辑器，支持编辑与预览
- **搜索**：暂不实现，后续按需添加

**笔记标题规则：**
- 新建笔记时不要求输入标题，默认为"无标题笔记"
- 列表中的显示名优先取：用户设定的标题 > 内容首行文本 > "无标题笔记"
- 用户可以随时在主窗口中为笔记补充或修改标题

### 2.2 便签窗口（Note Pad）— 快捷记录与磁贴的统一体

便签窗口是同一个窗口在不同生命周期阶段的不同形态，而非两个独立功能。

**完整生命周期：**

```
热键呼出 → 编辑内容 → 保存关闭（销毁）
                    ↘ 钉住为磁贴（常驻）→ 取消钉住 → 保存关闭（销毁）
```

**阶段一：快捷记录状态（呼出态）**

- 全局热键呼出，在任意场景下弹出一个轻量小窗
- **默认直接进入编辑区**，无需先填标题，光标直接就位可以开始写
- 支持两种模式切换：
  - **新建模式**（默认）：打开空白编辑区
  - **打开模式**：列出已有笔记供快速选择
- 编辑完成后，用户可以：
  - **保存并关闭**：内容保存为笔记，窗口销毁
  - **钉住**：窗口转变为磁贴状态，常驻屏幕
- 全局热键通过 Win32 API `RegisterHotKey` 实现

**阶段二：磁贴状态（钉住态）**

便签窗口被"钉住"后进入磁贴状态：

- 窗口缩小并贴到屏幕指定位置
- 可自由缩放大小、拖拽位置
- 内容持续可编辑
- 两种吸附模式：
  - **置顶悬浮**：始终在其他窗口之上（`Topmost`）
  - **贴在桌面**：嵌入桌面壁纸层之上、图标层之下（通过 `WorkerW` 窗口实现）
- 支持同时存在多个磁贴
- 关闭后不自动恢复（下次启动不记忆磁贴状态）
- 取消钉住后回到普通编辑状态，可再次保存关闭

**也可以从主窗口发起：** 在主窗口中对某条笔记执行"钉住"操作，直接创建该笔记的磁贴窗口。

### 2.4 系统集成

- **开机自启**：注册表写入或快捷方式放入 Startup 文件夹
- **系统托盘常驻**：最小化后驻留托盘，托盘菜单提供快捷操作
- **全局热键**：至少一个用于呼出快捷记录，后续可扩展更多

### 2.5 导入 / 导出

- **导入**：支持导入 `.md` 文件到笔记库
- **导出**：支持将笔记导出为 `.md` 文件
- 后续可考虑批量导入/导出

## 三、数据存储方案

### 3.1 笔记内容

- 每条笔记存储为独立的 `.md` 文件
- 存储目录由用户配置（默认为 `用户文档目录/花笺/notes/`）
- 文件名规则：`{id}_{标题}.md`（id 保证唯一性，标题保证可读性）；无标题时为 `{id}.md`

### 3.2 元数据索引

- 使用一个轻量索引文件（`metadata.json` 或 SQLite）管理笔记元信息
- 索引内容：

```json
{
  "notes": [
    {
      "id": "uuid",
      "title": "笔记标题",
      "fileName": "uuid_笔记标题.md",
      "createdAt": "2026-04-26T10:00:00",
      "updatedAt": "2026-04-26T12:00:00"
    }
  ]
}
```

- 索引用于快速加载笔记列表，避免每次启动都扫描文件系统
- 后续扩展标签、分类等功能时，只需在索引中增加字段，不影响 `.md` 文件本身

### 3.3 应用配置

- 存储为 `config.json`，位于应用数据目录
- 内容包括：全局热键设定、笔记存储路径、窗口位置记忆等

## 四、技术方案

### 4.1 项目结构（预设）

```
花笺/
├── 花笺.sln
├── src/
│   └── 花笺/
│       ├── App.xaml                    # 应用入口
│       ├── Models/                     # 数据模型
│       │   ├── Note.cs
│       │   └── AppConfig.cs
│       ├── ViewModels/                 # MVVM ViewModel
│       │   ├── MainViewModel.cs
│       │   ├── EditorViewModel.cs
│       │   └── NotePadViewModel.cs     # 便签窗口（快捷记录 + 磁贴统一）
│       ├── Views/                      # WPF 视图
│       │   ├── MainWindow.xaml
│       │   └── NotePadWindow.xaml      # 便签窗口（快捷记录 + 磁贴统一）
│       ├── Services/                   # 业务逻辑
│       │   ├── NoteService.cs          # 笔记 CRUD
│       │   ├── MetadataService.cs      # 元数据索引管理
│       │   ├── HotkeyService.cs        # 全局热键
│       │   ├── TrayService.cs          # 系统托盘
│       │   └── DesktopEmbedService.cs  # 桌面嵌入（WorkerW）
│       ├── Helpers/                    # 工具类
│       │   └── Win32Interop.cs         # P/Invoke 声明
│       └── Resources/                  # 资源文件、样式
└── README.md
```

### 4.2 关键技术点

| 功能 | 技术方案 |
|------|----------|
| MVVM 框架 | CommunityToolkit.Mvvm |
| Markdown 渲染 | Markdig（解析） + 自定义 WPF 渲染，或集成现有控件 |
| Markdown 编辑器 | AvalonEdit（代码编辑器控件，支持语法高亮） |
| 全局热键 | Win32 `RegisterHotKey` / `UnregisterHotKey` via P/Invoke |
| 系统托盘 | `NotifyIcon`（WPF 可用 Hardcodet.NotifyIcon.Wpf） |
| 桌面嵌入 | FindWindow 查找 `Progman` → 发送 `0x052C` → 枚举 `WorkerW` → SetParent |
| 开机自启 | 注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` |
| JSON 处理 | System.Text.Json |

### 4.3 架构模式

- **MVVM**：View ↔ ViewModel ↔ Model，WPF 标准模式
- **服务层**：ViewModel 通过 Service 类访问文件系统和系统 API，便于解耦和测试

## 五、开发阶段规划

**目标**：跑通最小闭环——能创建、编辑、保存一条笔记。

## 七、约束与原则

1. **轻量优先**：不引入不必要的依赖，保持启动速度
2. **文件透明**：用户的笔记始终是可读的 `.md` 文件，不锁定在私有格式中
3. **渐进式开发**：先跑通核心流程，再逐步添加功能，每个阶段都应是可用的
4. **本地优先**：所有数据存储在本地，无网络依赖
