# 第一部分：Markdown 写作指南

## 1. Markdown 基础入门

### 1.1 什么是 Markdown

Markdown 是一种**轻量级标记语言**，由 John Gruber 于2004年创建。它的核心理念是：

- **易读易写**：使用简单的符号标记格式，纯文本也能轻松阅读
- **专注内容**：不用在复杂的排版按钮中切换，双手不离键盘就能完成排版
- **跨平台兼容**：纯文本格式，任何编辑器都能打开，不会出现格式错乱

### 1.2 为什么在花笺中使用 Markdown

花笺原生支持 Markdown 渲染，为您带来：

1. **所见即所得的预览**：左侧写代码，右侧立即看到排版效果
2. **专业排版效果**：标题、列表、代码块、表格、数学公式一应俱全
3. **数据可移植**：您的笔记就是标准 `.md` 文件，可以在任何 Markdown 编辑器中打开
4. **纯文本存储**：文件体积极小，不依赖特定软件就能阅读

### 1.3 学习建议

- Markdown 语法非常简单，常用语法不到10个标记，10分钟就能学会
- 建议边学边练：在花笺中新建一篇笔记，跟着教程输入示例代码，立即在预览区看到效果
- 不必一次学完所有语法，先掌握基础的20%就能满足80%的日常需求

## 2. 基础语法详解

> 本章介绍最常用的 Markdown 语法，建议全部掌握。每个语法点都包含「写法」「效果」「说明」三部分，方便对照学习。

### 2.1 标题

标题用于组织文章结构，分为6个级别，用 `#` 号标记，`#` 数量代表级别。

**写法：**

```markdown
# 一级标题（最大）

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题（最小）
```

**效果：**

# 一级标题（最大）

## 二级标题

### 三级标题

#### 四级标题

##### 五级标题

###### 六级标题（最小）

**注意事项：**

- `#` 后必须加一个空格，否则不会被识别为标题
- 建议一篇笔记只有一个一级标题（作为笔记标题），正文从二级标题开始
- 花笺会自动为标题生成锚点，可用于文档内跳转

### 2.2 段落与换行

**段落：** 直接写文字就是段落，段落之间用空行分隔。

```markdown
这是第一个段落。

这是第二个段落。
（空一行表示新段落）
```

**换行：** 在一行末尾加**两个空格**然后回车，或者直接空一行。

```markdown
第一行（两个空格后回车）
第二行（紧接上一行）
```

**注意事项：**

- 普通回车不会产生换行，只会被当作空格处理
- 不要用空格缩进段落开头，Markdown 中缩进有特殊含义

### 2.3 文字强调

#### 粗体

用两个 `*` 或两个 `_` 包裹文字。

**写法：** `这是**粗体文字**效果`
**效果：** 这是**粗体文字**效果

#### 斜体

用一个 `*` 或一个 `_` 包裹文字。

**写法：** `这是*斜体文字*效果`
**效果：** 这是*斜体文字*效果

#### 粗斜体

用三个 `*` 包裹文字。

**写法：** `这是***粗斜体文字***效果`
**效果：** 这是***粗斜体文字***效果

#### 删除线（GFM 扩展，见第4章）

用两个 `~` 包裹文字。

**写法：** `这是~~删除线~~效果`
**效果：** 这是~~删除线~~效果

#### 高亮标记（花笺扩展）

用两个 `=` 包裹文字。

**写法：** `这是==高亮文字==效果`
**效果：** 这是==高亮文字==效果（需开启 HTML 渲染）

**花笺快捷操作：** 选中文字后点击工具栏的 **B**（粗体）或 _I_（斜体）按钮。

### 2.4 列表

#### 无序列表

用 `-`、`*` 或 `+` 加空格开头，效果相同。

**写法：**

```markdown
- 列表项一
- 列表项二
  - 子列表项（缩进2个空格）
  - 子列表项
- 列表项三
```

**效果：**

- 列表项一
- 列表项二
  - 子列表项（缩进2个空格）
  - 子列表项
- 列表项三

#### 有序列表

用数字加 `.` 加空格开头，数字顺序不影响最终渲染。

**写法：**

```markdown
1. 第一项
2. 第二项
   1. 子项一
   2. 子项二
3. 第三项
```

**效果：**

1. 第一项
2. 第二项
   1. 子项一
   2. 子项二
3. 第三项

**注意事项：**

- 标记符号后必须有空格
- 子列表需要缩进2个空格
- 列表项内可以包含段落、引用等其他元素，需要缩进对齐

### 2.5 引用

用 `>` 加空格开头表示引用，可以嵌套。

**写法：**

```markdown
> 这是一段引用文字。
> 可以多行连续使用。
>
> > 这是嵌套引用。
```

**效果：**

> 这是一段引用文字。
> 可以多行连续使用。
>
> > 这是嵌套引用。

**花笺快捷操作：** 选中文字后点击工具栏的引用按钮（引号图标）。

### 2.6 分割线

三个或更多的 `-`、`*` 或 `_` 单独占一行。

**写法：**

```markdown
---
---

---
```

**效果：** 三种写法效果相同，都是一条水平分割线：

---

### 2.7 链接

#### 行内链接

格式：`[链接文字](链接地址 "可选标题")`

**写法：**

```markdown
访问 [花笺官网](https://github.com/cheng806415/floral-notepaper) 获取更多信息。
```

**效果：** 访问 [花笺官网](https://github.com/cheng806415/floral-notepaper) 获取更多信息。

#### 自动链接

用 `<>` 包裹 URL 或邮箱地址。

**写法：** `<https://github.com>`
**效果：** <https://github.com>

**注意事项：**

- 花笺中点击外部链接会用系统默认浏览器打开
- 以 `#` 开头的链接会在当前文档内跳转（跳转到对应标题锚点）

### 2.8 图片

格式与链接类似，前面加一个 `!`：`![替代文字](图片路径)`

**写法：**

```markdown
![示例图片](images/example.png)
```

**花笺中的图片用法：**

- **直接粘贴**：在编辑器中按 `Ctrl+V` 粘贴剪贴板中的图片，图片会自动保存到笔记目录的 `images/` 文件夹
- **本地图片**：使用相对路径 `images/文件名.png` 引用（推荐）
- **网络图片**：直接使用完整 URL（不推荐，需要联网且可能失效）
- **拖拽插入**：将图片文件拖拽到编辑器中

图片在预览中默认以50%宽度居中显示。详细说明见第二部分第8章「图片功能」。

### 2.9 行内代码

用反引号 `` ` `` 包裹代码片段。

**写法：** ``在命令行输入 `npm install` 安装依赖``
**效果：** 在命令行输入 `npm install` 安装依赖

**花笺快捷操作：** 选中文字后点击工具栏的代码按钮（`</>` 图标）。

## 3. 高级语法

### 3.1 代码块

用于展示多行代码，支持语法高亮。用三个反引号 ` ``` ` 包裹，并在第一行反引号后指定语言（可选但推荐）。

**写法：**

````markdown
```javascript
function hello(name) {
  console.log(`Hello, ${name}!`);
  return true;
}
```
````

**效果：**

```javascript
function hello(name) {
  console.log(`Hello, ${name}!`);
  return true;
}
```

**支持的语言：** 花笺支持大多数常见编程语言的语法识别，包括但不限于：

| 语言       | 标记名              | 语言   | 标记名           |
| ---------- | ------------------- | ------ | ---------------- |
| JavaScript | `javascript` / `js` | Python | `python` / `py`  |
| TypeScript | `typescript` / `ts` | Rust   | `rust` / `rs`    |
| HTML       | `html`              | CSS    | `css`            |
| Java       | `java`              | C/C++  | `c` / `cpp`      |
| Go         | `go`                | Shell  | `bash` / `shell` |
| JSON       | `json`              | SQL    | `sql`            |
| Markdown   | `markdown`          | YAML   | `yaml` / `yml`   |

**花笺特性：**

- 代码块右上角会显示语言标签
- 鼠标悬停时显示「复制」按钮，点击可一键复制代码
- 代码块默认使用等宽字体，长代码自动横向滚动

### 3.2 表格

使用 `|` 分隔列，`-` 分隔表头和内容，`:` 控制对齐方式。

**基本写法：**

```markdown
| 姓名 | 年龄 | 职业   |
| ---- | ---- | ------ |
| 张三 | 25   | 工程师 |
| 李四 | 30   | 设计师 |
```

**效果：**

| 姓名 | 年龄 | 职业   |
| ---- | ---- | ------ |
| 张三 | 25   | 工程师 |
| 李四 | 30   | 设计师 |

**对齐方式：**

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 内容   |   内容   |   内容 |
```

- `:---` 左对齐（默认）
- `:---:` 居中
- `---:` 右对齐

**注意事项：**

- 表格前后需要空行
- 单元格内可以使用行内格式（粗体、斜体、代码等）
- 各列的 `|` 不需要严格对齐，只要数量正确即可

### 3.3 数学公式（LaTeX）

花笺支持 LaTeX 数学公式渲染，使用 KaTeX 引擎。

#### 行内公式

用单个 `$` 包裹公式，嵌入文字中。

**写法：** `质能方程：$E = mc^2$`
**效果：** 质能方程：$E = mc^2$

#### 块级公式

用两个 `$$` 包裹公式，单独成段居中显示。

**写法：**

```markdown
$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$
```

**效果：**

$$
\int_{-\infty}^{\infty} e^{-x^2} dx = \sqrt{\pi}
$$

#### 常用公式示例

**分数：** `\frac{a}{b}` → $\frac{a}{b}$

**上下标：** `x^2` → $x^2$，`x_i` → $x_i$

**根号：** `\sqrt{x}` → $\sqrt{x}$，`\sqrt[n]{x}` → $\sqrt[n]{x}$

**求和/积分：**

```markdown
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$

**矩阵：**

```markdown
$$
\begin{pmatrix}
a & b \\
c & d
\end{pmatrix}
$$
```

$$
\begin{pmatrix}
a & b \\
c & d
\end{pmatrix}
$$

**花笺快捷操作：** 点击工具栏的公式按钮可快速插入行内公式 `$ $` 或块级公式。

### 3.4 转义字符

如果想显示 Markdown 中具有特殊含义的字符（如 `*`、`#`、`[` 等），在前面加反斜杠 `\` 即可。

**写法：** `\*这不是斜体\*，\# 这不是标题`
**效果：** \*这不是斜体\*，\# 这不是标题

**需要转义的常见字符：**

| 字符 | 名称     | 字符    | 名称   |
| ---- | -------- | ------- | ------ | ------ |
| `\`  | 反斜杠   | `` ` `` | 反引号 |
| `*`  | 星号     | `_`     | 下划线 |
| `{}` | 花括号   | `[]`    | 方括号 |
| `()` | 圆括号   | `#`     | 井号   |
| `+`  | 加号     | `-`     | 减号   |
| `.`  | 英文句号 | `!`     | 感叹号 |
| `    | `        | 竖线    | `~`    | 波浪号 |
| `$`  | 美元符号 | `=`     | 等号   |

## 4. GFM 扩展语法（GitHub Flavored Markdown）

花笺完整支持 GitHub 风格的 Markdown 扩展语法（GFM）。

### 4.1 删除线

用两个波浪号 `~~` 包裹文字。

**写法：** `这是~~已删除~~的内容`
**效果：** 这是~~已删除~~的内容

### 4.2 任务列表（待办事项）

在无序列表基础上，使用 `[ ]`（未完成）或 `[x]`（已完成）标记任务状态。

**写法：**

```markdown
- [x] 已完成的任务
- [ ] 待完成的任务
- [ ] 计划中的任务
```

**效果：**

- [x] 已完成的任务
- [ ] 待完成的任务
- [ ] 计划中的任务

**注意事项：**

- `[ ]` 中括号内必须有一个空格
- 任务列表可以嵌套
- 花笺预览中复选框为展示效果，点击不会改变状态（编辑时手动修改 `[x]`/`[ ]`）

### 4.3 警告块（Alerts）

警告块是 GitHub 风格的特殊引用块，用于突出显示重要信息。基于引用语法，第一行使用特殊标记。

**写法：**

```markdown
> [!NOTE]
> 这是一条备注信息，用于补充说明。

> [!TIP]
> 这是一条提示，帮助用户更好地完成操作。

> [!IMPORTANT]
> 这是重要信息，用户需要特别留意。

> [!WARNING]
> 这是警告信息，提醒用户注意潜在风险。

> [!CAUTION]
> 这是注意事项，操作不当可能导致问题。
```

**效果：**

> [!NOTE]
> 这是一条备注信息，用于补充说明。

> [!TIP]
> 这是一条提示，帮助用户更好地完成操作。

> [!IMPORTANT]
> 这是重要信息，用户需要特别留意。

> [!WARNING]
> 这是警告信息，提醒用户注意潜在风险。

> [!CAUTION]
> 这是注意事项，操作不当可能导致问题。

**五种类型区别：**

| 类型      | 标记           | 使用场景           |
| --------- | -------------- | ------------------ |
| NOTE      | `[!NOTE]`      | 补充说明、额外信息 |
| TIP       | `[!TIP]`       | 小技巧、更好的做法 |
| IMPORTANT | `[!IMPORTANT]` | 关键信息、必须了解 |
| WARNING   | `[!WARNING]`   | 潜在风险、需要警惕 |
| CAUTION   | `[!CAUTION]`   | 负面后果、操作警示 |

### 4.4 自动链接识别

GFM 会自动识别文本中的 URL 和邮箱地址，不需要手动加 `<>`。

**写法：** `访问 https://github.com 或联系 support@example.com`
**效果：** 访问 https://github.com 或联系 support@example.com（自动转为可点击链接）

## 5. HTML 标签支持

> 注意：此功能需要在花笺设置中开启「允许 HTML 标签渲染」选项，默认关闭。

花笺在开启 HTML 渲染后，支持部分安全的 HTML 标签，用于实现 Markdown 语法无法表达的样式。

### 5.1 支持的 HTML 标签

| 标签        | 用途          | 示例                                                 |
| ----------- | ------------- | ---------------------------------------------------- |
| `<mark>`    | 高亮文字      | `<mark>重点内容</mark>`                              |
| `<u>`       | 下划线        | `<u>带下划线的文字</u>`                              |
| `<center>`  | 居中对齐      | `<center>居中文字</center>`                          |
| `<font>`    | 字体颜色/大小 | `<font color="red">红色文字</font>`                  |
| `<abbr>`    | 缩写          | `<abbr title="World Health Organization">WHO</abbr>` |
| `<br>`      | 强制换行      | `第一行<br>第二行`                                   |
| `<sub>`     | 下标          | `H<sub>2</sub>O`                                     |
| `<sup>`     | 上标          | `x<sup>2</sup>`                                      |
| `<details>` | 折叠块        | 见下方示例                                           |

### 5.2 折叠块示例

```html
<details>
  <summary>点击展开查看详情</summary>

  这里是折叠的内容，可以包含任意 Markdown 语法。 - 列表项 - **粗体文字**
</details>
```

**效果：**

<details>
<summary>点击展开查看详情</summary>

这里是折叠的内容，可以包含任意 Markdown 语法。

- 列表项
- **粗体文字**

</details>

### 5.3 上标下标示例

**写法：**

```markdown
水的化学式是 H<sub>2</sub>O，光速约为 3×10<sup>8</sup> m/s。
```

**效果：** 水的化学式是 H<sub>2</sub>O，光速约为 3×10<sup>8</sup> m/s。

### 5.4 安全说明

花笺使用了 HTML 净化（sanitize）机制，会自动过滤危险标签（如 `<script>`、`<iframe>` 等）和危险属性，防止 XSS 攻击。仅允许白名单内的安全标签和属性。

## 3. 笔记基础操作

### 3.1 新建笔记

**方法一：按钮新建**

1. 点击侧边栏顶部的「+」按钮
2. 新笔记会自动创建在当前选中的分类下
3. 在标题栏输入笔记标题（可留空，会显示为「无标题笔记」）
4. 在编辑区开始写作

**方法二：快捷键新建**
在主窗口中按 `Ctrl+N` 快速新建笔记。

> 提示：笔记标题就是 Markdown 文件的文件名。标题中的特殊字符（如 `/ \ : * ? " < > |`）会自动替换为安全字符。

### 3.2 编辑笔记

1. 在侧边栏笔记列表中点击要编辑的笔记
2. 笔记内容加载到编辑器
3. 直接在编辑区修改内容
4. 修改后状态栏显示「未保存」

**自动保存：** 默认开启，停止输入约1秒后自动保存。您也可以通过 `Ctrl+S` 手动保存。

### 3.3 保存笔记

**自动保存：**

- 主窗口笔记默认开启自动保存
- 小窗/磁贴笔记可在设置中独立配置自动保存
- 外部文件（直接打开的 `.md`）也可配置自动保存

**手动保存：** 按 `Ctrl+S` 立即保存。

**保存状态说明：**

| 状态     | 含义                     |
| -------- | ------------------------ |
| 已保存   | 所有更改已写入磁盘       |
| 未保存   | 有未保存的修改           |
| 保存中   | 正在写入磁盘             |
| 保存失败 | 写入出错，请检查文件权限 |

### 3.4 删除笔记

1. 在侧边栏笔记列表中右键点击要删除的笔记
2. 选择「删除笔记」
3. 在确认对话框中点击「确认删除」

> 注意：删除操作不可恢复！笔记文件会被直接删除到系统回收站，如果误删请从回收站恢复。

### 3.5 搜索笔记

1. 点击侧边栏顶部的搜索框
2. 输入关键词，实时过滤笔记列表
3. 搜索范围包括笔记标题和内容
4. 点击搜索框右侧的「x」清空搜索，显示全部笔记

## 8. 图片功能

花笺支持在笔记中插入本地图片，图片会被保存到笔记目录下的 `images/` 文件夹。

### 8.1 插入图片

**方法一：粘贴（最常用）**

1. 复制图片到剪贴板（截图、网页图片右键复制等）
2. 在编辑器中光标定位到要插入图片的位置
3. 按 `Ctrl+V` 粘贴
4. 图片自动保存并插入 Markdown 图片语法

**方法二：拖拽插入**

1. 从文件管理器中将图片文件拖拽到编辑器中
2. 松开鼠标，图片自动复制并插入

**方法三：手动输入语法**

```markdown
![图片描述](images/文件名.png)
```

### 8.2 图片存储

- 图片保存在笔记目录的 `images/` 子文件夹中
- 文件名使用随机生成的唯一名称，避免重名冲突
- 支持的格式：PNG、JPG/JPEG、GIF、WebP、SVG 等常见格式
- 单张图片大小上限：20 MB

### 8.3 清理未使用图片

当您删除笔记中引用的图片语法后，图片文件仍保留在 `images/` 文件夹中。定期清理可释放磁盘空间：

1. 当笔记包含 `images/` 引用时，状态栏右侧会出现「清理未使用图片」按钮
2. 点击该按钮，系统扫描哪些图片文件不再被任何笔记引用
3. 确认后删除未使用的图片
4. 状态栏显示清理结果（已清理 X 张图片 / 没有需要清理的图片）

### 8.4 图片显示

- 预览区图片默认以50%宽度居中显示
- 支持懒加载（滚动到图片位置才加载）
- 背景图片（见设置章节）可自定义主窗口背景

## 9. 导入与导出

### 9.1 导入 Markdown 文件

1. 点击侧边栏的「导入 Markdown」按钮
2. 在文件选择对话框中选择一个或多个 `.md` 文件
3. 文件被复制到笔记目录，并添加到笔记列表
4. 导入的笔记归类到当前选中的分类下

> 注意：仅支持 `.md` / `.markdown` / `.txt` 格式文件。导入时文件中的图片引用不会自动导入，需要手动将图片放到 `images/` 文件夹。

### 9.2 导出笔记

1. 在侧边栏笔记列表中右键点击要导出的笔记
2. 选择「导出 Markdown」
3. 选择保存位置
4. 笔记内容保存为 `.md` 文件

> 注意：导出仅导出 Markdown 文本内容，图片不会一起导出。如需备份全部数据，请直接备份整个笔记目录。

### 9.3 外部文件编辑

除了导入到笔记库，您也可以直接编辑外部 Markdown 文件：

**Windows：** 右键 `.md` 文件 → 打开方式 → 选择花笺
**macOS/Linux：** 将 `.md` 文件的默认打开方式设为花笺

外部文件特点：

- 文件保存在原位置，不复制到笔记目录
- 出现在「外部文件」分类下
- 可配置是否自动保存
- 关闭时有未保存更改会提示保存

### 9.4 数据备份

花笺所有数据保存在笔记目录中（默认位置：`用户文档/floral-notepaper/`）。备份整个文件夹即可完整备份所有笔记、图片和配置。

配置文件位置：

- Windows: `%AppData%/com.floral-notepaper/`
- macOS: `~/Library/Application Support/com.floral-notepaper/`
- Linux: `~/.config/com.floral-notepaper/`

## 10. 设置详解

点击标题栏的齿轮图标打开设置面板。

### 10.1 通用设置

**语言**

- 支持简体中文、繁體中文、English
- 切换语言后界面立即生效

**主题**

- 浅色：明亮主题
- 深色：暗色主题
- 跟随系统：根据系统主题自动切换

**笔记目录**

- 显示当前笔记存储路径
- 点击「选择文件夹」可更改笔记目录
- 更改目录后需要手动迁移已有笔记文件

**开机自启**

- 开启后系统启动时自动运行花笺
- 默认最小化到托盘

**关闭到托盘**

- 开启后点击关闭按钮不退出程序，最小化到系统托盘
- 托盘图标右键可完全退出或显示窗口

### 10.2 编辑器设置

**默认视图**

- 选择启动时默认使用编辑/分栏/预览哪种模式
- 推荐新用户选择「分栏」

**编辑器字号**

- 主编辑器的字体大小（单位：像素）
- 范围：12-24px，默认14px

**小窗/磁贴字号**

- 快捷便签和磁贴窗口的字号
- 独立于主编辑器设置

**Tab 缩进宽度**

- 按 Tab 键插入的空格数
- 可选 2、4、8

**自动保存笔记**

- 主窗口笔记编辑后自动保存
- 建议开启，防止意外丢失内容

**小窗笔记自动保存**

- 小窗/磁贴编辑后自动保存

**外部文件自动保存**

- 打开的外部 `.md` 文件是否自动保存
- 谨慎开启，避免意外修改原文件

**编辑预览同步滚动**

- 分栏模式下左右同步滚动
- 默认开启

### 10.3 快捷键设置

**快捷记录快捷键**

- 全局快捷键，按下后呼出快捷便签小窗
- 点击「点击录制」按钮后按下想要的组合键
- 必须包含 Ctrl、Alt 或 Command/Win 修饰键
- 按 Delete 键可清除快捷键
- 系统会检测快捷键冲突并提示

**显示/隐藏窗口快捷键**

- 全局快捷键，快速显示或隐藏主窗口
- 不能与快捷记录快捷键重复

### 10.4 窗口设置

**记住小窗尺寸**

- 记住调整后的小窗和磁贴窗口大小
- 关闭后每次打开使用默认尺寸

**快捷键打开时跟随鼠标位置**

- 开启后快捷便签在鼠标位置弹出
- 关闭后在屏幕中心显示

**Ctrl+右键快速关闭磁贴**

- 按住 Ctrl 右键点击磁贴快速关闭
- 适合磁贴较多时快速清理

### 10.5 磁贴设置

**磁贴渲染 Markdown**

- 磁贴窗口是否渲染 Markdown 格式
- 关闭则显示纯文本，性能更好

**磁贴颜色**

- 跟随主题：磁贴颜色随浅色/深色主题变化
- 自定义：选择固定颜色作为磁贴背景

### 10.6 Markdown 设置

**允许 HTML 标签渲染**

- 开启后支持 `<mark>`、`<u>`、`<center>`、`<font>` 等HTML标签
- 默认关闭，开启后请注意不要打开不可信来源的 Markdown 文件

### 10.7 背景图片设置

支持自定义主窗口背景图片：

- **默认背景**：使用应用内置背景
- **选择图片**：选择本地图片作为背景
- **清除背景**：恢复默认背景
- **填充模式**：
  - 填充：覆盖整个窗口（可能裁剪）
  - 完整：完整显示图片（可能有留白）
  - 平铺：重复平铺图片
- **遮罩**：调整背景遮罩透明度，控制内容可读性
- **模糊**：对背景图片应用高斯模糊
- **缩放**：调整背景图片缩放比例
- **横向/纵向位置**：调整背景图片的位置

# 第三部分：常见问题与技巧

## 常见问题 FAQ

### 安装与启动问题

**Q: Windows SmartScreen 提示「Windows已保护你的电脑」怎么办？**
A: 这是因为应用未购买代码签名证书。点击「更多信息」→「仍要运行」即可正常启动。

**Q: macOS 提示「无法打开，因为无法验证开发者」怎么办？**
A: 在「系统设置 → 隐私与安全性」中，找到被阻止的应用提示，点击「仍要打开」。或右键应用图标选择「打开」。

**Q: 开机自启不生效？**
A: Windows 请检查是否被安全软件拦截了启动项；macOS 请在「系统设置 → 通用 → 登录项」中确认花笺已添加。

**Q: 全局快捷键没反应？**
A:

1. 检查是否被其他软件占用了相同快捷键（常见冲突：输入法切换、其他启动器类软件）
2. 尝试在设置中更换其他快捷键组合
3. 快捷键必须包含 Ctrl/Alt/Command/Win 修饰键

### 数据存储问题

**Q: 我的笔记保存在哪里？**
A: 默认保存在「用户文档\floral-notepaper」文件夹中。每篇笔记是一个独立的 `.md` 文件，图片存放在 `images/` 子文件夹。可以在设置中更改笔记目录。

**Q: 如何在多台电脑间同步笔记？**
A: 花笺不内置云同步，但您可以：

1. 将笔记目录设置在 OneDrive、Dropbox、iCloud Drive、坚果云等同步盘目录中
2. 使用 Git 手动版本管理笔记目录
3. 定期手动备份笔记文件夹

**Q: 重装系统前如何备份？**
A: 备份两个地方：

1. 整个笔记目录（包含所有 .md 文件和 images 文件夹）
2. 配置文件目录（见第9.4节路径）

**Q: 笔记文件可以用其他编辑器打开吗？**
A: 完全可以！笔记就是标准 Markdown 格式的纯文本文件，任何文本编辑器（VS Code、Typora、Obsidian 等）都能打开编辑。但请注意：用其他编辑器修改后，花笺中的自动保存可能会覆盖外部修改。

### 编辑问题

**Q: 粘贴图片失败怎么办？**
A:

1. 检查图片格式是否支持（PNG/JPG/GIF/WebP）
2. 检查单个图片是否超过 20MB
3. 检查笔记目录的写入权限
4. 尝试先将图片保存为文件再拖拽插入

**Q: 为什么预览区图片不显示？**
A:

1. 确认图片路径为 `images/文件名` 格式（相对路径）
2. 确认图片文件确实存在于笔记目录的 `images/` 文件夹中
3. 网络图片需要联网才能加载

**Q: 数学公式不渲染怎么办？**
A:

1. 确认公式语法正确（LaTeX 语法）
2. 行内公式用单个 `$` 包裹，块级公式用两个 `$$` 包裹
3. `$` 符号内侧不要有空格（部分情况下可能导致识别问题）

**Q: 同步滚动不准确怎么办？**
A: 同步滚动基于滚动高度比例，并非逐行精确对齐（这是 Markdown 预览的通用方案，因为编辑区和预览区行高不同）。如果需要精确位置，可手动滚动微调。

**Q: HTML 标签不生效？**
A: 需要在设置中开启「允许 HTML 标签渲染」选项（默认关闭，出于安全考虑）。

### 磁贴/小窗问题

**Q: 磁贴无法置顶？**
A: Linux 部分窗口管理器可能不支持窗口置顶；Windows/macOS 应该正常工作。

**Q: 磁贴太多怎么快速清理？**
A: 在设置中开启「Ctrl+右键快速关闭磁贴」，然后按住 Ctrl 键右键点击想要关闭的磁贴即可快速关闭。

**Q: 小窗大小记不住？**
A: 请在设置中开启「记住小窗尺寸」选项。

### 导入导出问题

**Q: 导入 Markdown 后图片不显示？**
A: 导入功能只复制 .md 文件，不会自动下载或导入图片。需要手动将图片文件放到笔记目录的 `images/` 文件夹中，并确保文件路径正确。

**Q: 导出笔记时图片会一起导出吗？**
A: 目前导出仅导出 Markdown 文本。需要图片请手动复制 `images/` 文件夹中的对应图片。

## 使用技巧与注意事项

### 效率提升技巧

**1. 快速记录工作流**

- 设置一个顺手的全局快捷键（如 `Ctrl+;` 或 `Alt+N`）
- 灵感来了随时按快捷键呼出小窗
- 输入完成后点击关闭按钮，内容会自动保存
- 需要继续编辑时，点击小窗右上角「在编辑器中打开」按钮转到主编辑器

**2. 善用分类组织笔记**

- 创建「工作」「学习」「生活」「想法」等分类
- 配合分类快速切换不同场景
- 定期整理未分类笔记

**3. 充分利用 Markdown 语法**

- 写待办清单用任务列表 `- [ ]`
- 重要内容用警告块 `> [!IMPORTANT]`
- 数学笔记直接插入 LaTeX 公式
- 代码片段用代码块并指定语言获得语法高亮

**4. 分栏模式最佳实践**

- 默认使用分栏模式，边写边看效果
- 开启同步滚动保持视觉对齐
- 长文档写作时切换到纯编辑模式减少干扰
- 最终阅读/检查时切换到纯预览模式

**5. 磁贴便签工作流**

- 把待办清单钉在桌面上作为提醒
- 把常用参考资料（如命令速查表、常用链接）钉在桌面随时查看
- 临时任务记在小窗，结束后转为磁贴或删除

**6. 图片管理**

- 直接 Ctrl+V 粘贴截图最方便
- 定期使用「清理未使用图片」释放磁盘空间
- 重要图片建议备份到其他位置

### 写作建议

**1. 标题层级规范**

- 一篇笔记建议只有一个一级标题（通常就是笔记标题，编辑器标题栏已经有了，正文中可以从二级标题开始）
- 不要跳级使用标题（比如从 ## 直接到 ####）
- 合理使用标题层级让文档结构清晰

**2. 段落间距**

- 段落之间空一行，阅读体验更好
- 列表项内多段内容需要适当缩进

**3. 代码块习惯**

- 代码块始终指定语言，获得语法高亮
- 长代码考虑拆分或折叠显示（使用 details 标签）

**4. 链接管理**

- 引用网络资源时插入超链接而不是直接贴 URL
- 重要本地文档可以使用相对路径互相引用

### 注意事项

**安全相关**

- 不要开启「允许 HTML 标签渲染」后打开不可信来源的 Markdown 文件
- 花笺不会上传您的任何数据到网络，所有内容保存在本地
- 定期备份笔记目录，防止意外丢失

**性能相关**

- 单篇笔记不建议超过 10 万字（可能影响渲染性能）
- 大量图片会增加磁盘占用，建议定期清理
- 磁贴/小窗数量建议不超过 20 个同时打开

**兼容性相关**

- 笔记文件使用 UTF-8 编码，跨平台通用
- 行尾换行符自动适配操作系统
- 所有 Markdown 语法遵循 CommonMark + GFM 标准，可在其他编辑器中正常显示

### 从其他笔记应用迁移

**从 Typora 迁移：**

- Typora 的笔记就是标准 Markdown 文件，直接复制到花笺笔记目录即可
- 图片如果使用相对路径，需要将图片文件夹一并复制
- 通过「导入 Markdown」批量导入

**从 Obsidian 迁移：**

- Obsidian 也是 Markdown 文件，直接复制 .md 文件和 attachments 文件夹
- 将图片放到花笺的 images 文件夹，并更新图片路径为 `images/文件名`
- Wiki 链接 `[[笔记名]]` 需要手动改为标准 Markdown 链接

**从 印象笔记/Notion 迁移：**

- 先导出为 Markdown 格式（Notion 支持导出为 Markdown，印象笔记需要借助工具）
- 再通过导入功能导入
- 注意图片资源路径需要手动调整
